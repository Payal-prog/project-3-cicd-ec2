Project Overview

This project demonstrates a containerized CI/CD pipeline that builds, versions, and deploys a web application to AWS infrastructure. The deployment is automated through GitHub Actions and served through a Dockerized Nginx container.

Key Features

  • GitHub Actions CI/CD pipeline
  • Docker image build and push to Docker Hub
  • Image tagging using latest + Git commit SHA
  • Application Load Balancer health checks
  • Auto Scaling Group instance refresh deployment
  • Container auto-restart policy

Deployment Flow

Every code change triggers a pipeline that builds a new Docker image, publishes it to Docker Hub, and refreshes the Auto Scaling Group so new instances pull and run the latest version.

Architecture

  • Application Load Balancer as the public entry point
  • Auto Scaling Group launches EC2 instances from a Launch Template
  • EC2 user data installs Docker and runs the container
  • Nginx serves the application inside Docker
  • Health checks available at /health.html

Security Design

  • Public users access only the ALB DNS endpoint
  • EC2 application port is restricted to ALB security group traffic
  • AWS credentials are stored securely in GitHub Secrets
  • Docker Hub token is used instead of account password
