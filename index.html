Automated Docker-based deployment on AWS using GitHub Actions

Project Overview

This application demonstrates a fully automated CI/CD pipeline that builds, versions, and deploys a containerized web application to AWS infrastructure.

Deployment Flow

  • Code pushed to GitHub
  • GitHub Actions builds Docker image
  • Image tagged with latest + commit SHA
  • Pushed to Docker Hub
  • Deployed to AWS EC2 via SSH
  • Application served through Application Load Balancer

Architecture

  • Application Load Balancer (public access)
  • EC2 instance (private, secured)
  • Docker container (Nginx)
  • Health checks via /health.html

Key Features

  • Automated CI/CD pipeline
  • Dockerized deployment
  • Versioned images (Git SHA)
  • Manual + auto workflow triggers
  • Container auto-restart policy
  • Secure access via ALB (no direct EC2 exposure)

Deployment Metadata

  • Deployment Type: Containerized CI/CD
  • Image Registry: Docker Hub
  • Web Server: Nginx inside Docker
  • Traffic Path: ALB → EC2:8080 → Container:80
  • Health Endpoint: /health.html
